## Prototypes Changes, data.lua
- global.lua.  All globals are now warp inside an module to prevent collision. See zerg's global.lua for example.
```lua
-- from
local name = MOD_NAME .. '--drone--2' 
-- to
local name = ERM_ZERG.MOD_NAME .. '--drone--2'
```
- category attributes in recipe prototypes need to change to categories,  And need to be a table value. Example:
```lua
-- from
category = "chemistry",
-- to
categories = {"chemistry"},
```
- loot changed to use the item prototype. Example:
```lua
            unit_prototype.loot = {
                {type = "item", name = organ_name, independent_probability = 0.33,  amount_min = multiplier[1], amount_max = multiplier[2]}
            }
```
- "probability" property in item prototype changed to "independent_probability"
- lightning damage change from number to damage prototype "{ amount = 500 , type = "electric" }"

    data.extend {

```

## Runtime changes, control.lua
- entity.minable became read-only. Change to entity.minable_flag for write.
- Feature group's point has changed to use multiplier, instead points 
```lua
--- OLD
race_settings.featured_groups = {
    -- Unit list, spawn ratio, unit attack point multiplier 
    { { "zealot", "dragoon" }, { 6, 3 }, 25 },
    { { "zealot", "archon" }, { 6, 3 }, 50 },
    { { "zealot", "dragoon", "archon", "reaver" }, { 4, 3, 2, 1 }, 35},
}

--- NEW
race_settings.featured_groups = {
    -- Unit list, spawn ratio, unit attack point multiplier 
    -- Formular: 10 * GROUP_SIZE * Map Setting Multiplier * Multiplier Deviation * This multiplier
    -- By default:  10 * 200 * 1.25 * random(0.8 - 1.2) * 1.1
    { { "zealot", "dragoon" }, { 6, 3 }, 1.1 },
    { { "zealot", "archon" }, { 6, 3 }, 0.9 },
    { { "zealot", "dragoon", "archon", "reaver" }, { 4, 3, 2, 1 }, 0.9 },
}
```
